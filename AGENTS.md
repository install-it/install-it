## Project

Windows PC setup assistant / driver installer. Wails v2 desktop app: Go backend + Vue 3 frontend.

## Commands

```sh
# Dev (both Go + Vue hot-reload)
wails dev

# Build production binary
wails build

# Build with version stamp
wails build -ldflags "-X main.buildVersion=<version>"

# Go tests (pkg only â€” root package tests need Wails runtime)
go test ./pkg/...

# Frontend tests
cd frontend && npm test

# Frontend lint + format
cd frontend && npm run lint
cd frontend && npm run format

# Frontend type-check only
cd frontend && npm run type-check
```

## Architecture

- **Go backend** (`main.go`, `app.go`, `pkg/`): Wails-bound structs exposed to frontend via `Bind`. Entry point is `main()` â€” it opens SQLite, runs migrations, then starts Wails runtime.
- **Vue frontend** (`frontend/src/`): Pages, components, Pinia stores, composables, i18n. Uses Nuxt UI v4 + TailwindCSS v4.
- **Wails bridge** (`frontend/wailsjs/`): Auto-generated Goâ†’JS bindings. **Do not edit** â€” regenerated by `wails dev` / `wails build`. ESLint and Prettier ignore this directory.
- **SQLite** (`pkg/storage/`): CGO-free driver (`glebarez/sqlite`). Schema migrations via `gormigrate/v2`. Baseline migration guards legacy installs.
- **App settings**: JSON file (`conf/setting.json`), not in SQLite. Uses `FileStore`/`MemoryStore`.
- **Updater** (`pkg/update/`): Native Go self-update using Windows file-rename trick. Python updater is a **separate repo** (`install-it-updater`) â€” don't look for it here.

## Key constraints

- **Windows-only app**: Go code uses Windows-specific APIs (file rename trick, WMI, WebView2). Build targets are x86 and x64.
- **CGO-free**: SQLite driver is pure Go. No CGO required for cross-compilation.
- **`init()` side effects**: `main.go` `init()` creates `conf/` and `drivers/` directories next to the binary. Root-level tests (`app_test.go`) note this is harmless.
- **`TriggerNativeUpdate` is untestable end-to-end**: It calls `os.Exit(0)`; only `extractBinaryFromZip` is unit-tested.
- **Porter closes DB on import**: `porter.Porter` calls `db.Close()` before backup and `db.Reopen()` + `db.Migrate()` after import. If editing storage, account for this lifecycle.
- **Wails bindings must be in `Bind` list**: New Go structs exposed to frontend must be added to `Bind` in `main.go`. Enums go in `EnumBind`.

## CI

- **Test workflow** (`.github/workflows/test.yml`): Runs on PRs to `main`. Executes `go test ./pkg/...` on Windows.
- **Release workflow** (`.github/workflows/build_and_release.yml`): Triggered by `v*` tags. Builds x86 + x64 Windows binaries with UPX compression, bundles WebView2 runtime, builds Python updater from separate repo, drafts GitHub release.
